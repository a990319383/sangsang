> 敏感数据脱敏展示需求


## 二、方案对比

| 对比项 | 序列化脱敏 | **field-encryptor** |
|:---:|:---:|:---:|
| **脱敏阶段** | JSON 序列化层 | **SQL 执行层** |
| **使用方式** | 接口返回类标注 | **SQL 返回类标注** |
| **业务差异化** | 需额外配置 | **原生支持** |
| **适用范围** | 单字段 / 整体对象 | **单字段 / 整体对象 / Map / String** |

### 示例对比

> 场景：A 类型订单的物料需要脱敏，其他类型不需要

| 方案 | 实现复杂度 |
|:---:|:---:|
| 序列化脱敏 | 需要额外配置差异化规则 |
| **field-encryptor** | **获取完整对象，直接判断业务类型** |

## 三、工作原理

> 框架基于 MyBatis 拦截器，对 SQL 执行结果集进行脱敏处理

```
┌─────────────────────────────────────────────────────┐
│                                                          │
│  业务代码执行 SQL 查询                                 │
│         ↓                                                │
│  MyBatis 拦截器拦截                                   │
│         ↓                                                │
│  识别需要脱敏的字段和策略                               │
│         ↓                                                │
│  对结果集进行脱敏处理                                     │
│         ↓                                                │
│  输出脱敏后的结果                                       │
│                                                          │
└─────────────────────────────────────────────────────┘
```

### 效果示例

> 场景：订单列表中 A 类型的物料需要脱敏

**原始 SQL 查询结果**
```
| 订单号 | 订单类型 | 物料名称       |
|--------|----------|--------------|
| 001     | A        | 敏感物料名字   |
| 002     | A        | 敏感物料名字   |
| 003     | B        | 普通物料名字   |
```

**自动脱敏后**
```
| 订单号 | 订单类型 | 物料名称       |
|--------|----------|--------------|
| 001     | A        | ***脱敏物料*** |
| 002     | A        | ***脱敏物料*** |
| 003     | B        | 普通物料名字   |
```

## 四、支持的返回类型

| 返回类型 | 标注方式 | 说明 |
|:---:|:---|:---|
| **实体类** | 字段上标注 `@FieldDesensitize` | 针对单个字段脱敏 |
| **String** | Mapper 上标注 `@MapperDesensitize` | 返回字符串直接脱敏 |
| **Map** | Mapper 上标注 `@MapperDesensitize` | 指定字段名脱敏 |

```properties

```

### Step 3: 自定义脱敏策略

```java
package xxx.strategy;

import com.sangsang.domain.strategy.desensitize.DesensitizeStrategy;

/**
 * 业务差异化脱敏策略示例
 */
public class BusinessDesensitizeStrategy implements DesensitizeStrategy {

    @Override
    public String desensitize(String s, Object o) {
        // s: 待脱敏的字符串（可能为 null）
        // o: 整个返回对象，可获取业务信息进行差异化脱敏

        if (o instanceof OrderVo) {
            OrderVo order = (OrderVo) o;
            // A 类型的订单进行物料脱敏
            if ("A".equals(order.getOrderType())) {
                return desensitizeMaterial(s);
            }
        }

        // 其他情况不脱敏
        return s;
    }

    /**
     * 物料名称脱敏逻辑
     */
    private String desensitizeMaterial(String material) {
        if (material == null) {
            return null;
        }
        if (material.length() <= 3) {
            return material.charAt(0) + "***";
        }
        return material.substring(0, 2) + "***";
    }
}
```

### Step 4: 字段标注

> **⚠️ 注意**
>
> • 这是标注在 **SQL 返回类**上，不是实体类
>
> • 区别于框架的其他功能

#### 响应是实体类

```java
/**
 * SQL 返回的 VO 类
 */
public class OrderVo {
    private Long orderId;
    private String orderType;

    /**
     * 物料名称 - 需要脱敏
     */
    @FieldDesensitize(BusinessDesensitizeStrategy.class)
    private String materialName;
}
```

#### 响应是 String

```java
/**
 * 订单 Mapper
 */
public interface OrderMapper {

    /**
     * 查询订单名称列表 - 返回 String 需要脱敏
     */
    @MapperDesensitize(@FieldDesensitize(BusinessDesensitizeStrategy.class))
    List<String> getOrderNames(String orderId);
}
```

#### 响应是 Map

```java
/**
 * 订单 Mapper
 */
public interface OrderMapper {

    /**
     * 查询订单详情 - Map 返回值脱敏
     */
    @MapperDesensitize({
        @FieldDesensitize(
            value = BusinessDesensitizeStrategy.class,
            fieldName = "material_name"  // SQL 结果集中的字段名（AS 后的别名）
        ),
        @FieldDesensitize(
            value = BusinessDesensitizeStrategy.class,
            fieldName = "order_desc"
        )
    })
    Map<String, Object> getOrderDetail(String orderId);
}
```

### 业务差异化脱敏示例

> 场景：根据用户权限、订单类型、数据敏感等级等条件进行差异化脱敏

```java
@Component
public class ComplexDesensitizeStrategy implements DesensitizeStrategy {

    @Override
    public String desensitize(String s, Object o) {
        if (s == null) {
            return null;
        }

        // 获取当前登录用户
        UserContext user = UserContext.getCurrentUser();

        // 管理员不脱敏
        if (user.isAdmin()) {
            return s;
        }

        // 普通用户根据数据敏感等级脱敏
        return desensitizeByLevel(s, getDataLevel(o));
    }

    private String desensitizeByLevel(String value, String level) {
        switch (level) {
            case "HIGH":
                return "***";
            case "MEDIUM":
                return value.substring(0, 1) + "***";
            default:
                return value;
        }
    }
}
```

## 八、常见问题

### Q1: 与序列化脱敏有什么区别？

| 对比项 | 序列化脱敏 | **SQL 层脱敏** |
|:---:|:---|:---:|
| **脱敏阶段** | JSON 序列化 | **SQL 执行后** |
| **使用位置** | 接口返回类 | **SQL 返回类** |
| **业务差异化** | 需额外配置 | **原生支持** |
| **适用场景** | Web 接口 | **所有场景** |

> **💡 提示**：如果项目已使用序列化脱敏，建议统一使用一种方案，避免冲突

---

### Q2: fieldName 参数如何指定？

> `fieldName` 指定的是 SQL 结果集中的变量名

| SQL 语法 | fieldName 值 |
|:---|:---|
| `SELECT name FROM table` | `"name"` |
| `SELECT name AS user_name FROM table` | `"user_name"` |
| `SELECT concat(first, last) AS full_name FROM table` | `"full_name"` |

---

### Q3: 如何实现不同字段使用不同脱敏策略？

```java
public class UserVo {

    // 物料脱敏策略
    @FieldDesensitize(MaterialDesensitizeStrategy.class)
    private String materialName;

    // 手机号脱敏策略
    @FieldDesensitize(PhoneDesensitizeStrategy.class)
    private String phoneNumber;
}
```
