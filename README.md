# field-encryptor

## 一个对业务代码零侵入的透明数据治理框架

[![Version](https://img.shields.io/badge/version-3.6.1--alpha-blue)](https://gitee.com/tired-of-the-water/field-encryptor)
[![License](https://img.shields.io/badge/license-Apache%202.0-blue)](LICENSE)
[![Java](https://img.shields.io/badge/Java-1.8%2B-orange)](https://www.oracle.com/java/)

> **旨在打造一个对业务代码零侵入的透明中间层**
>
> 基于 MyBatis 拦截器，通过动态 SQL 解析引擎
>
> 0侵入解决某些业务场景，一处标注，全局生效
>
> 区别于常规的解决方案，告别满项目标注

---

## 核心特性

| 特性 | 描述               | 简介                                        | 长期维护 |
| :--- |:-----------------| ------------------------------------------- | :------- |
| 🔐    | 字段加解密            | 敏感字段自动插入加密，查询解密              | ✅        |
| 🔄    | 异构数据库语法转换        | 不同数据库的语法自动切换，如mysql自动转达梦 | ✅        |
| 🛡️    | 自动数据隔离           | sql查询时自动拼接上当前数据隔离的where条件  | ✅        |
| 📝    | 数据变更字段自动填充       | 数据发生变更，自动维护本条数据的某些字段值  | ✅        |
| 🎭    | 数据库字段查询脱敏        | sql查询获取到的字段自动脱敏                 | ❌        |

------

## 核心优势

- **零侵入** - 基于注解配置，无需修改业务代码。所有controller，service，mapper，入参响应层都不需要改动
- **全局生效** - 一处标注，全项目应用。告别常规的满项目注解标注
- **灵活扩展** - 支持自定义策略和算法
- **性能优化** - LRU 缓存解析结果
